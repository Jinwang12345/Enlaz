## Code Decision Map

### Models Strategy
- **Auto-generated**: `Backend/models.py` (regenerated each run)
- **Custom models**: `backend/models.py` (manually maintained)
- **DTOs**: `backend/routes.py` (UserResponse, custom logic)

### Path Handling
```python
# ALWAYS use Path() from pathlib
from pathlib import Path

# ✅ CORRECT
openapi_file = Path(__file__).parent / "openapi" / "base.yaml"

# ❌ WRONG
openapi_file = os.path.join("Backend", "openapi", "base.yaml")
```

### Response Models
```python
# Input validation (strict)
class User(LaiaBaseModel):
    email: str = Field(...)
    # ...errors rejected here

# Output serialization (lenient)
class UserResponse(BaseModel):
    email: Optional[str] = Field(None)
    # ...missing fields OK
```

## Avoid Common Pitfalls

❌ **Don't**:
- Edit `Backend/` auto-generated files directly (they get overwritten)
- Use `os.path.join()` for paths (use `Path()` instead)
- Assume field validation rules (check current schema first)
- Ignore engram context (always load before proceeding)

✅ **Do**:
- Update `openapi.yaml` when changing schemas
- Use `backend/` directory for custom code
- Subscribe to context.project_key for state changes
- Save progress to engram at session end

## Troubleshooting

**"ImportError: UserResponse not found"**
→ Check `backend/routes.py` exists and has UserResponse class

**"FileNotFoundError: base.yaml"**
→ Use `Path(__file__).parent` instead of relative paths

**"Validation error: Field required"**
→ Check model is using right class (User vs UserResponse)

**"Models keep reverting"**
→ Normal behavior (LAIA regenerates). Edit `backend/routes.py` custom logic instead.
